- 覆盖四块：异常控制流总体概念、异常（exceptions）、进程（processes）、进程控制（创建/等待/执行/退出）。

## 为什么需要“异常控制流”（ECF）

- 仅靠条件跳转与调用/返回无法对“系统状态的变化”及时响应（如磁盘/网络到达、除零、Ctrl-C、时钟中断），因此需要一套从指令流中“跳出去”的机制——异常控制流（ECF）。
- ECF 贯穿系统各层：低层有“异常”；高层有“进程上下文切换”“信号”“非局部跳转”等。

## 异常（Exceptions）

- 本质：因事件触发而把控制权转交给内核；处理后可能返回当前/下一条指令，或终止程序。
- 分类：
  - **异步**（外部设备触发的中断），处理后返回下一条指令，如定时器、I/O 到达、键盘等。
  - **同步**：由指令执行导致
    - **Trap**（有意触发，如系统调用），返回下一条。
    - **Fault**（可能可恢复，如缺页），处理后重执行当前指令；若不可恢复则终止。
    - **Abort**（不可恢复，如非法指令/硬件错误），直接终止。
- 示例：缺页异常把页面从磁盘载入后重放指令；非法地址导致内核向进程发送 SIGSEGV 并终止 。
- **系统调用**：用户态通过 `syscall` 切到内核态，约定参数/返回寄存器；返回值负数对应 errno 。

## 进程（Processes）

- 定义：一个**正在运行的程序的实例**；为每个程序提供两大抽象：
  1. 逻辑控制流（“好像独占 CPU”）由**上下文切换**提供；
  2. 私有地址空间（“好像独占内存”）由**虚拟内存**提供。
- 多道程序：单核下各进程的控制流在时间上交错并发；多核可真正并行。
- **上下文切换**：内核代码并非独立进程，而是运行在现有进程上下文中；切换时保存/恢复寄存器、切换地址空间 。

## 进程控制（创建/执行/等待/退出）

- **创建**：`fork()` 被调用一次却返回两次——在父进程返回子 PID、在子进程返回 0；子进程拥有与父几乎相同但彼此独立的地址空间与打开文件表项副本（PID 不同）。
  - 经典示例展示父子并发与“各自独立的 x 值”。
- **执行新程序**：`execve(filename, argv, envp)` 用新程序映像覆盖当前进程（保留 PID、打开文件和信号上下文），正常情况下**不返回**；常见用法是子进程 `fork` 后 `execve` 运行命令 。
- **等待/回收**：父进程用 `wait`/`waitpid` 获取子进程结束状态并**回收（reap）**资源；否则子进程会成为“僵尸（defunct）”，直到被回收或被 init(1) 接管回收。
- **退出**：`exit(status)` 或从 `main` 返回导致进程终止；某些信号的默认动作也会终止进程。
- **健壮性**：系统调用出错通常返回 -1 并设置 `errno`，需检查每次系统调用的返回值（示例给出 `fork` 错误处理与封装）。
