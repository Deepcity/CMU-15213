assignments:

1. 列出所有会在组1命中的缓存。
2. 列出所有会在组6命中的缓存

## Answer

- **字节寻址**；一次访问的是 **1 字节**。
- **地址宽度**：13 位。
- **2 路组相联**：$E=2$（每组两条行/两路）。
- **块大小**：$B=4$ 字节 ⇒ **块内偏移位数** $b=\log_2 4=2$。
- **组数**：$S=8$ ⇒ **组索引位数** $s=\log_2 8=3$。
- **标记位数**：$t=13-s-b=13-3-2=8$（正好 2 个十六进制数）。

> 结论：
>  **地址分解**（高到低）：`[ tag(8) | set index(3) | block offset(2) ]`
>
> - **组号** = 地址的中间 3 位
> - **块内字节偏移** = 地址最低 2 位 ∈ {0,1,2,3}
> - **标记** = 最高 8 位

### 如何用表格做一次访问判断

给定一个 13 位地址（十六进制更常见）：

1. **写成整数**，按位分出 `tag / set / offset` 三段。
2. 到表中对应的 **“组索引”那一行**，看 **行0** 和 **行1**：
   - 只在 **有效位=1** 的那条行上比较 **标记**；
   - 若标记相等 ⇒ **命中**，返回该行中 **“字节offset”** 列里的那个字节；
   - 两路都没匹配 ⇒ **未命中**（miss）。
3. 未命中时如何替换取决于策略（LRU/随机等）。题面通常不考替换，或会明确给出策略；若没给，就只判“miss”。

> 小技巧：从**表中已有的行**，你还能**反推会命中的所有地址**。
>  只要地址的 `set` 与该行的组号一致、`tag` 与该行的标记一致，`offset` 取 0~3 中任意一个，都会命中；且返回的字节分别是“字节0~3”。

### 组1与组6命中的缓存

- **组1**：`set index = 001`，命中的地址有：
  - `tag = 0x45`，`offset = 00/01/10/11`
- **组6**：`set index = 011`，命中的地址有：
  - `tag = 0x45`，`offset = 00/01/10/11`

```sh
# 组1地址可能是

01000101(45) 001 xx

# format 

0 1000 1010 01xx
# 0x08A4 - 0x08A7
```

对组1相联的另一个行则是：0x08A4 - 0x08A7。

对于组6则是只有tag为0x91的地址命中，地址范围为0x0704-0x0707。

## 附录

这里省略了一些知识点

1. 看到书上说“写可以命中，但读出位置”，十有八九是在说：**写未命中（miss）时，如果采用 write-allocate，会把要写的块放进这个组里一个 `valid=0` 的路上**。它“命中”的只是**放置位置**（placement），**不是数据命中**。更准确的表述应是：**写未命中，但选中了一个空路进行分配**。

2. bit offset偏移的下标应该是从零开始计算的
