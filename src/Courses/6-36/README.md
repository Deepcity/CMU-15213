# 6-36

![6-36](https://s2.loli.net/2025/11/06/ENf85ybFHKm47LU.png)

这是一个综合考察的题目，若熟悉缓存的工作原理，可以跳过。

```cpp
int x[2][128];
for (i = 0; i < 128; i++) 
    sum += x[0][i] * x[1][i];
```

## Answer

### A. 512B，直接映射，块 16B：不命中率？

- 512B / 16B = **32 行**（direct-mapped 32 线）。
- 两个流的 **行号相同**：`(b1 mod 32) = (b0+32) mod 32 = b0 mod 32`
   ⇒ `x[0][i]` 和 `x[1][i]` 每次都映射到**同一行**，而且按顺序交替访问。
- 访问顺序：先读 `x[0][i]`，再读 `x[1][i]`。由于是两块不同的块，同一行上会**彼此顶掉**：
   每一次访问都是 miss（冷/冲突抖动）。

**Miss 数** = 256；**miss rate = 256/256 = 100%**。

### B. 把高速缓存容量翻倍到 1024B（仍直接映射、块 16B）：不命中率？

- 1024B / 16B = **64 行**。
- `b1 mod 64 = (b0+32) mod 64 ≠ b0 mod 64` → 两个流现在落在**不同的行**，互不干扰。
- 每个连续 4 个元素共用一块：**每 4 次只在第一次 miss，后面 3 次 hit**。
   每行（128 元素）块数 `128/4 = 32`，所以每个流 32 次 miss。
- 两个流合计 miss = 32+32 = 64。

**miss rate = 64 / 256 = 25%**。

### C. 回到 512B，但改为**2 路组相联 + LRU**，块 16B：不命中率？

- 512B / 16B = 32 行；2 路 ⇒ **16 组 × 每组 2 路**。
- 两个流的块都映射到**同一组**（因为 `32 mod 16 = 0`），但**两路可同时驻留**，互不顶替。
- 仍然只有“每块第一次”的**强制不命中**：每流 32 块 → 64 次 miss。

**miss rate 仍是 64 / 256 = 25%**。

### D.（基于情形 C）增大**缓存容量**能否进一步降低不命中率？

**不能（或效果不明显）**。
 情形 C 的 25% 全是**强制不命中**（每新块第一次访问）。工作集在任一时刻只需要 **2 个块**（两个行当前块），2 路已能容纳，不存在容量/冲突不命中。增大容量不会减少强制不命中。

------

### E.（基于情形 C）增大**块大小**能否降低不命中率？为什么？

**可以**（在总容量固定为 512B 的前提下，2 路仍足以同时容纳两行的当前块）。
 原因：顺序访问同一行具有**空间局部性**，块更大 ⇒ **每块覆盖更多元素** ⇒ 每行需要的块数减少 ⇒ **强制不命中减少**。

定量：若块大小为 `B`，每行块数 `512/B`，两行合计 miss `2×(512/B)`，
 总访问 256 次，**miss rate = (1024/B)/256 = 4/B**。

- 当 `B=16`：`4/16 = 25%`（与上面一致）

- 当 `B=32`：`4/32 = 12.5%`

…直到 `B=256`：`4/256 ≈ 1.56%`（此时仅 1 组 × 2 路，也能同时放下两个块，仍无冲突）
